# Projects

Projects are at the core of Code for Boston. Our projects vary widely, but all of them have a few things in common:

- They serve a real, documented civic need (we're not in the business of creating solutions in search of problems).
- We work with an external partner that has the ability to take over ownership of the project when we're done building.
- We don't build mission-critical infrastructure. Our projects are designed to be proofs of concept, nice-to-haves, experiments, or v1.0 of a longer-term priority our project partner will own.
- Our projects are open source and mission aligned.

## Typical project structure:

All projects start with a project lead (a Code for Boston volunteer) and an external project sponsor. In some rare cases this can be the same person, but only if the external partner has the capacity to act as a consistent project lead.  

Most projects evolve to include multiple sub-teams, often with their own leads. These might include:
- Front end developers
- Back end developers
- Design and user research
- Data + research
- And more!

The "average" Code for Boston project takes about 18 months from project kickoff to v1.0 launch, though this can vary quite a lot! Some of our projects continue to be built, improved, and maintained by Code for Boston, but only if there is interest and voluteer capacity to keep working. If there isn't, the project is handed over to the project partner after launch for them to own and operate.  

## How to Choose a Project

After Orientation, feel free to sit in on one or more project team meetings to see if the topic is interesting to you, you like the people on the team, and if you can contribute in a way that will fulfil whatever goals you have coming into Code for Boston.  

Ask yourself:

- Does the mission interest me?
- Can I contribute with my current skills?
- Will I learn something new? (You're not at work! You can use this time to learn something new, not just do the same thing you do in your day job.)
- Do I enjoy spending time with the other people on the project?

## Where do new projects come from?

Most project ideas are submitted using the [New Project Idea submission form linked at the bottom of this page](https://codeforboston.org/projects/). Occasionally, the Code for Boston core team will solicit projects from our contacts in Local and State government, or from local community-based organizations. If you have an idea but you're not sure where to start, the Code for Boston core team is always happy to help refine ideas and point you in the right direction!
